1. Регистрация
1.1	Пользователь даёт команду на регистрацию.
1.2	Сайт отображает форму регистрации.
	1.2.1	Создать шаблон страницы
1.3	Пользователь заполняет поля формы (уникальная почта, ник, пароль) и подтверждает регистрацию.
	1.3.1	Добавить поле для ввода уникальной почты
	1.3.2	Добавить поле для ввода ника
	1.3.3	Добавить поле для ввода пароля
	1.3.4 	Кнопка подтверждения регистрации
1.4	Сайт подтверждает правильность заполнения формы(все поля не пустые ,не больше указанного размера, не состоят только из пробелов и введённой почты не существует в БД). Если некорректный ввод, то выводится соответствующее сообщение и поля очищаются. Выход из сценария.
1.5	Если пользователь существует - выбрасываем исключение. Выход из сценария. Иначе хешируем пароль.
1.6	Добавление в БД нового пользователя.

2. Вход в систему
2.1	Сайт отображает форму входа.
	2.1.1 Создать шаблон страницы входа
2.2	Если пользователь зарегистрирован, то см. пункт 3, если нет, то дальше он переходит на сценарий регистрации.
2.3	Пользователь вводит почту и пароль и нажимает "Войти".
	2.3.1	Добавить поле ввода почты
	2.3.2	Добавить поле ввода пароля
	2.3.3	Кнопка "Войти"
2.4	Сайт проверяет введенные данные на корректность(все поля не пустые, не больше указанного размера). Если некорректный ввод, то выводится соответствующее сообщение и поля очищаются. Выход из сценария.
2.5	Сайт сверяет данные с базой.
	2.5.1	Поиск пользователя, если не найдет, то сообщение. Выход из сценария.
	2.5.2	Хешируем пароль
	2.5.3	Выборка по логину и паролю. 
2.6	Если такой пользователь с таким паролем есть в базе(результат выборки непустой), то пользователь переходит на главную страницу, online=true. Если нет, то выводится сообщение о том, что такой пользователь не найден.

3. Написание и добавление поста
3.1	Пользователь даёт команду на написание поста(нажимает “Say” на верхней панели или использует специальные поля на новостной странице).
3.2	Сайт отображает поля для ввода сообщения и добавления тегов.
3.3	Пользователь вводит сообщение, вводит теги(про теги подробнее в пункте «Добавление тегов»), нажимает “Say it”.
3.4	Сайт проверяет введенные данные на корректность(все поля не пустые, не больше указанного размера). Если некорректный ввод, то выводится соответствующее сообщение и поле для тегов очищается. Выход из сценария.
3.5	Сообщение добавляется в БД и появляется на странице юзера и в новостях тех пользователей, которые на него подписаны.

4. Добавление тегов
4.1	Пользователь находится в окошке для написания постов. Слева от поля для текста поста есть поле для тегов и кнопка «Добавить тег».
	4.1.1	Рядом с полем для поста добавить окошко для ввода тегов и кнопку "Добавить"
4.2	Пользователь вводит тег. Нажимает на кнопку «Добавить тег».
4.3	Сайт проверяет введенные данные на корректность(все поля не пустые, не больше указанного размера). Если некорректный ввод, то выводится соответствующее сообщение и поле для тегов очищается. Выход из сценария. 
4.4	Если такой тег есть, то увеличивается счетчик популярности тега, если нет, то тег добавляется в БД. 
4.5	Тег появляется в овале рядом с сообщением.
4.6	Очищается поле для тегов. Если пользователь хочет добавить ещё теги, то переход на пункт 2, если нет, то конец.

5. Поиск постов по тегам
5.1	Сайт отображает форму поиска(при нажатии на “Interesting” на верхней панели).
	5.1.1	Создать шаблон страницы Интерестного
5.2	Пользователь вводит в текстовое поле название искомого тега, нажимает «добавить».
	5.2.1	Текстовое поле для ввода тегов
	5.2.2	Кнопка "Добавить"
5.3	Сайт проверяет введенные данные на корректность(все поля не пустые, не больше указанного размера). Если некорректный ввод, то выводится соответствующее сообщение и поле для тегов очищается. Выход из сценария. 
5.4	Если тег существует, то выводятся посты с этим тегом, отсортированные по популярности(количество лайков от большего к меньшему). Если нет, то выводится сообщение о том, что посты с данным тегом ещё не были созданы.
	5.4.1	Вывод постов с введенным тегом
	5.4.2	Сортировка по популярности
5.5	Пользователь может выбрать сортировку постов (A-Z, Z-A, по популярности, по дате).
	5.5.1	Выпадающий список для выбора сортировки
	5.5.2	Сортировка A-Z
	5.5.3	Сортировка Z-А
	5.5.4	Сортировка по дате
5.6	Если пользователь хочет добавить тег, то переход на пункт 2. В результате на странице будут выведены сообщения по всем искомым тегам, отсортированные по выбранному способу.

6. Пользователь ставит лайк
6.1	Пользователь находится на странице, где он может поставить лайк(новости, страница другого пользователя, своя страница или поиск постов по тегам).
	6.1.1	Кнопка для лайка под каждым постом
	6.1.2	Счетчик лайков рядом с кнопкой
6.2	Пользователь нажимает на сердечко.
6.3	Счетчик лайков поста увеличивается на 1.
6.4	Устанавливается true в булевскую переменную, которая отображает поставил ли данный пользователь лайк данному посту.
6.5	Пользователь не может поставить лайк ещё раз на пост, где он ставил лайк.

7. Пользователь подписывается на другого пользователя
7.1	Пользователь находится на странице пользователя, на которого хочет быть подписан.
	7.1.1	Создать шаблон страницы другого пользователя
7.2	Нажимает на кнопку подписаться.
	7.2.1	Кнопка "Подписаться"
7.3	В БД устанавливаются определенные связи.
7.4	На странице вместо кнопки подписаться появляется кнопка «отписаться».
7.5	В результате интересный пользователь будет в списке подписок и его посты будут отображаться в ленте новостей.

8. Блокировка пользователя
8.1	Пользователь находится на странице своих подписчиков.
8.2	Нажимает на кнопку «Заблокировать» напротив ника блокируемого пользователя.
	8.2.1 Кнопка Заблокировать напротив каждого подписчика
8.3	В БД устанавливаются определенные связи (отменяются взаимные подписки, если они имеются и добавление связки в таблицу с блоком).
8.4	В результате заблокированный пользователь не может видеть посты заблокировавшего в новостях, а при переходе к нему на страницу видит только его фотографию и имя (никаких постов). Не может на него подписаться.

9. Изменение настроек
9.1	Сайт отображает страницу настроек(пользователь нажал на кнопку Settings на левой панели).
	9.1.1	Добавить шаблон страницы настроек
9.2	Если пользователь хочет изменить ник/инфу о себе, то он вводит новое значение в определенном поле  и нажимает кнопку Сохранить изменения.
	9.2.1 	Поле для изменения ника
	9.2.2	Поле для изменения инфы о себе
	9.2.3	Кнопка "Сохранить изменения", которая применима ко всем изменениям на странице
9.3	Если пользователь хочет поменять почту, то детальнее в «смена почты».
9.4	Если хочет поменять пароль, то детальнее в «смена пароля».
9.5	Если хочет добавить/изменить фотографию, то он нажимает “Upload” для загрузки фотографии и Сохранить изменения.
	9.5.1	Кнопка upload
	9.5.2 	Логика кнопки upload
9.6	Если пользователь хочет сделать тему ночной, то он переставляет качельку “night mode” в true и нажимает сохранить изменения.
	9.6.1	Добавить качельку для тем
	9.6.2	Добавить темную тему
9.7	Если пользователь хочет удалить страницу, то детальнее в «удаление аккаунта».

10. Смена почты
10.1	Пользователь находится в меню настроек и нажимает «изменить почту».
	10.1.1	Кнопка "Изменить почту"
10.2	Сайт отображает форму смены почты.
	10.2.1	Добавить шаблон формы смены почты
10.3	Первоначально пользователь должен ввести адрес старой почты(адрес старой почты показывается на лейбле над полем ввода и отображается следующим образом: видны только первые три буквы адреса, все остальное - звездочки). Затем нажать на кнопку "Подтвердить". На форме также отображаются поле для ввода новой почты и поле для ввода присланного кода, но они неактивны.
	10.3.1	Поле для ввода адреса старой почты
	10.3.2	Кнопка "Подтвердить"
	10.3.3	Неактивное поле для ввода нового адреса
	10.3.4	Неактивная кнопка "Отослать код"
	10.3.5	Неактивное поле для ввода кода
	10.3.6	Неактивная кнопка "Подтвердить код"
10.4	Проводится проверка корректности ввода.
10.5	Сверяется введенное значение со значением из БД. Если совпадает, то переход к пункту 6, если нет, то поле ввода очищается и переход к пункту 3.
10.6	Поле для ввода нового адреса и кнопка "Отослать код" становятся активными, а поле для ввода адреса старой почты и кнопка "Подтвердить" - неактивными.
10.7	Пользователь вводит почту и нажимает кнопку "Отослать код".
10.8	Проводится проверка корректности ввода.
10.9	На введенный адрес высылается код подтверждения.
10.10	Поле для ввода нового адреса и кнопка "Отослать код" становятся неактивными, а поле для ввода кода и кнопка "Подтвердить код" - активными.
10.11	Пользователь должен ввести код в соответствующем поле и нажать «подтвердить код».
10.12	Проводится проверка корректности ввода.
10.13	Если код правильный, то меняется привязка к почте.

11. Смена пароля
11.1	Пользователь находится в меню настроек и нажимает «изменить пароль».
11.2	Сайт отображает форму смены пароля. На форме есть поле для ввода кода подтверждения и кнопка "Подтвердить код", неактивное поле ввода нового пароля и подтверждение нового пароля, неактивная кнопка "Изменить".
	11.2.1	Создать шаблон формы смены пароля.
11.3	На привязанную почту высылается код подтверждения.
11.4	Пользователь вводит код подтверждения.
11.5	Проводится проверка корректности ввода.
11.6	Если код введен неверно, то соответствующее сообщение и очищение поля. Иначе поле для ввода кода подтверждения и кнопка "Подтвердить код" становятся неактивными, а поле ввода нового пароля, поле подтверждения нового пароля и кнопка "Изменить" - активными.
11.7	Пользователь вводит новый пароль, подтверждает его (сверка на идентичность двух строк) и нажимает «изменить». Если строки не совпадают, то соответствующее сообщение и очистка полей ввода, остаёмся на пункте 11.7.
11.8	Проводится проверка корректности ввода.
11.9	Пароль изменяется в БД.
11.10	Переход на страницу настроек.

12. Удаление аккаунта
12.1	Пользователь находится на странице настроек и выбирает «Удалить аккаунт».
	12.1.1	Кнопка "Удалить аккаунт".
12.2	Сайт показывает модальное окно для подтверждения («Вы точно хотите удалить страничку?»).
12.3	Если пользователь нажимает «Да», то сайт отображает форму подтверждения удаления. Если «Нет», то возврат на форму настроек. Выход из сценария
	12.3.1	Форма подтверждения удаления
12.4	Пользователю высылается код подтверждения на привязанную почту и этот код нужно ввести в поле на форме сайта.
	12.4.1	Поле ввода кода подтверждения
	12.4.2	Кнопка "Подтвердить"
12.5	Проводится проверка корректности ввода.
12.6	Если код совпадает с отправленным, то страница считается замороженной. Пользователь не сможет войти на неё, но все посты остаются.

13. Переход на страницу подписчиков/подписок
13.1	Пользователь нажимает “followed”/”following” соответственно.
13.2	Сайт отображает форму подписчиков/подписок.
	13.2.1	Создать шаблон формы подписчиков
	13.2.2	Создать шаблон формы подписок
13.3	Если нет ни одного подписчика, то сообщение об этом.

14. Выход из системы
14.1	Пользователь нажимает “log out” на левой панели.
14.2	Сайт отображает форму входа. Online=false.

15. Переход на личную страничку
15.1	Пользователь нажимает на свой юзернейм на левой панели.
15.2	Сайт отображает форму личной страницы.

16. Переход на страничку другого пользователя
16.1	Пользователь нажимает на юзернейм интересного пользователя (из новостей, интересного, подписок, подписчиков).
16.2	Сайт отображает форму выбранного пользователя.

17. Отписка от пользователя
17.1	Пользователь находится в своих подписках и нажимает на "Отписаться" напротив имени человека, на которого он больше не хочет быть подписан.
17.2	Изменения в БД(удаление связи между ними).
17.3	Пользователь больше не видит постов пользователя, от которого он отписался, в своей ленте новостей

18. Отправка фидбека
18.1 Пользователь нажимает на кнопку фидбека на главном меню-баре.
18.2 Сайт отображает форму фидбэка.
18.3 Пользователь вводит свой адрес почты и свое сообщение об ошибке.
18.4 Проверка корректности ввода.
18.5 Происходит отправка сообщения об ошибке на почту администрации.
